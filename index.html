Grammar.jj breakdown

Token : Defines the final language symbols
Example : Digit from 0 to 9, operators (multiply, divide, plus, minus) 
 TOKEN : /* OPERATORS */
{
  < PLUS : "+" >
| < MINUS : "-" >
| < MULTIPLY : "*" >
| < DIVIDE : "/" >
}

TOKEN :
{
  < CONSTANT : (< DIGIT >)+ >
| < #DIGIT : [ "0"-"9" ] >
}
Rule : Defines tokens arrangements 
void unary() :
{
  (x = < MINUS > )? element()
}
Equals to : 
 <UNARY>::== [<MINUS>] <ELEMENT>
Additional action : 
void unary() :
{Token x = null;}
{
  (x = < MINUS > )? element()
  {
    int a = pile.pop();
    if(x != null){
      a = -a;
    }
    pile.push(a);
  }
}

Composition breakdown

JavaCC read the .jj and generate java files used to read and translate the source files.
Class Parser : defines the function to read step by step the source file.
Class Token : define the source file's words used in the parser.
And others : error messages, exceptions, etc.

But what about our interpreter

A Stack to manage the numbers we use. Each execution of a rule use the stack to feed or empty it. 
void sum() :
{Token x;}
{
  term()
  (  (  x = < PLUS >  |  x = < MINUS > ) term()
  {
      int a,b;
      a = pile.pop();
      b = pile.pop();
      if(x.kind == PLUS) {pile.push(b+a);}
      else{pile.push(b-a);}
    }
  )*
}
